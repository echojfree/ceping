# 选品侦探（2.5D货架情报局）迭代方案

## 1. 目标与范围

- 目标：新增一个“游戏化+测评化”模块，让学生在有限时间内完成选品分析、风险识别与可验证假设提交。
- 范围：仅做加法迭代，不破坏现有模块（创意先锋、前线战壕、数据中枢）。
- 入口：Hub 新增“选品侦探局”按钮，进入独立场景屏幕。

## 2. 玩法流程（本次实现）

1. Scene 1（货架对比）  
   从3个候选品中选择主推品，读取基础指标（售价/成本/物流/投放/退货率）。

2. Scene 2（利润估算）  
   按所选商品估算单单利润并给出风险等级与理由。

3. Scene 3（评论情报）  
   读取用户评论样本，判断当前最优先处理的风险类型。

4. Scene 4（假设提交）  
   输出“选品假设 + 验证指标 + 测试方案”，强调可验证性。

5. 结算  
   生成本地测评结果并进入现有通用报告页。

## 3. 测评点设计（对应你的要求）

- 分析深度：是否能结合成本、转化、退货、评论证据做决策。
- 风险意识：是否优先识别高影响风险并提出控制动作。
- 假设可验证性：是否给出可量化指标和可执行测试方案（含对照/时间窗）。

## 4. 技术实现策略

- 新增前端本地模块状态 `productDetective`（不依赖新增后端题库，避免大改）。
- 复用现有：
  - 统一切屏机制；
  - 2.5D/3D 教练流式点评面板；
  - 通用报告页 `screen-report-generic`。
- 新增本地评分器：每幕返回
  - 反馈等级（excellent/good/poor）；
  - 技能增量（analysis/risk_control/experiment/planning 等）；
  - RIASEC 增量（I/C/E 为主）。

## 5. 产出与验收

- 产出：
  - Hub 新入口；
  - 新屏幕 `screen-product-detective`；
  - 新流程（4幕）；
  - 本地结算结果映射到报告页；
  - E2E 自动化用例。
- 验收：
  - 可完整从入口打到报告；
  - 反馈文本与技能条可见；
  - 不影响现有 3 个实战模块和岗位情报功能。
